# 🚗 CarMagazine

**Версия:** 1.0.2  
**Язык:** Java 11+  
**Лицензия:** WTFPL Version 2  
**Авторы:** DariaBurd, fnJey, Nastya211296 , ayfSan, alxmkhl
**Репозиторий:** [GitHub](https://github.com/DariaBurd/CarMagazine/)

Консольное приложение для управления коллекцией автомобилей с поддержкой поиска, сортировки, многопоточности и тестирования.

---

## 📋 Описание

**CarMagazine** — это система управления базой данных автомобилей с цветным консольным интерфейсом. Приложение позволяет загружать данные из файлов, вводить вручную или генерировать случайные записи. Предоставляет возможности поиска по различным критериям, сортировки коллекции, экспорта результатов и встроенного тестирования.

---

## ✨ Возможности

| Функция | Описание |
|---------|----------|
| 📥 **Загрузка данных** | Из файла (.txt), ручной ввод, случайная генерация |
| 🔍 **Поиск** | По цвету, комплектации, мощности (диапазон), году выпуска |
| 🔄 **Сортировка** | По всем полям + специальное задание (чётные/нечётные годы) |
| 💾 **Экспорт** | Сохранение результатов поиска, сортировки и всей базы в файл |
| 🧵 **Многопоточность** | Выполнение тяжёлых задач в отдельном потоке (вкл/выкл) |
| 🧪 **Тестирование** | Встроенные тесты для автомобилей, поиска и сортировки |
| 🎨 **Цветной интерфейс** | ANSI-цвета для удобной навигации |
| 📊 **Кастомная коллекция** | Собственная реализация Collection с поддержкой стримов |

## 🏗 Архитектура и Паттерны

| Паттерн | Применение | Классы |
|---------|------------|--------|
| **Builder** | Валидация при создании объектов | `Automobile.Builder` |
| **Singleton** | Единое хранилище коллекции | `CarStorage` |
| **Strategy** | Различные алгоритмы поиска и сортировки | `SearchStrategy`, `SortStrategy` |
| **State** | Управление состояниями меню | `MenuState`, `MenuHandler` |
| **Factory** | Создание автомобилей разными способами | `CarFactory` |
| **Command** | Задачи для многопоточного выполнения | `Runnable` в `ThreadManager` |

## 🚀 Быстрый Старт

### Требования
- **Java:** 11+ (рекомендуется 17+)
- **Консоль:** с поддержкой ANSI-цветов
- **JUnit 5:** для запуска тестов

### Компиляция и Запуск

```bash
# Компиляция всех файлов
javac -d out src/com/carmagazine/**/*.java

# Запуск приложения
java -cp out com.carmagazine.Main
```

### Через IDE (IntelliJ IDEA / Eclipse)
1. Импортируйте проект
2. Убедитесь, что все классы в пакете `com.carmagazine`
3. Запустите `Main.java`

## 📄 Формат Файла

Файл с автомобилями должен содержать строки формата:
```
мощность,год,комплектация,цвет
```

**Пример (cars.txt):**
```
150,2020,Base,Белый
250,2022,Sport,Красный
350,2023,Premium,Черный
```

## 🔧 Технические Детали

### Многопоточность
- **ThreadManager** управляет `ExecutorService` (пул из 3 потоков)
- Тяжёлые операции (поиск, сортировка, загрузка) выполняются асинхронно
- Режим можно переключать через главное меню (пункт 4)

```java
// Включение/выключение многопоточности
threadManager.toggle();

// Выполнение задачи
threadManager.execute(() -> {
    // тяжёлая операция
});
```

### Валидация
```java
// Автомобиль с некорректными данными вызовет исключение
new Automobile.Builder()
    .setPower(-100)  // IllegalArgumentException
    .setYear(1500)   // IllegalArgumentException
    .build();
```

### Кастомная Коллекция
`CarCollection` расширяет `AbstractCollection`:
- Поддержка итераторов
- Стримы (`stream()`, `parallelStream()`)
- Интеграция со стратегиями сортировки

```java
CarCollection collection = new CarCollection();
collection.add(automobile);
collection.sortWith(new SortByYear());
```

### Стратегии Поиска
```java
SearchContext context = new SearchContext();
context.setStrategy(new SearchByPowerRange());
List<Automobile> result = context.executeSearch(cars, "100-200");
```

### Стратегии Сортировки
```java
AutomobileSort sorter = new AutomobileSort();
sorter.setStrategy(new SortByPower());
List<Automobile> sorted = sorter.sort(cars);
```

## ⚠️ Известные Ограничения

1. **ANSI-цвета** могут не отображаться в некоторых терминалах (Windows CMD)
2. **Кодировка файлов** должна быть UTF-8 для корректного отображения кириллицы
3. **Потокобезопасность** коллекции не гарантируется при параллельной модификации
4. В коде присутствуют **синтаксические ошибки** (пробелы в строках, опечатки), требующие исправления перед компиляцией
5. **Импорты** в некоторых файлах требуют исправления (звёздочки вместо конкретных импортов)
