# package fileSaver 
— сохранение результатов в файл.

## Информация о пакете
**Класс FileSaver в пакете fileSaver** реализует дополнительное задание 2 — запись отсортированных коллекций и найденных значений в файл в режиме добавления данных (append).

## Основные возможности:
- **Метод saveSortedCars()** — сохраняет отсортированную коллекцию автомобилей с указанием даты, типа сортировки и количества записей.
- **Метод saveSearchResults()** — сохраняет результаты поиска с указанием даты, типа поиска, поискового запроса и количества записей.

## Особенности реализации:
- Использование try-with-resources для автоматического закрытия потоков.
- Режим append (FileWriter(fileName, true)) — данные добавляются в конец файла, не перезаписывая существующие.
- Информативное форматирование: дата, заголовки, нумерация, разделители.
- Корректная обработка пустых коллекций.
- Обработка ошибок ввода-вывода с понятными сообщениями.

## Интеграция FileSaver в проекте
**В классе AutomobileSort (сортировка):**
- После выполнения сортировки пользователю предлагается сохранить результат.
- Метод save() вызывает FileSaver.saveSortedCars(), который передает название файла, отсортированную коллекцию и тип сортировки.
- Результат сохраняется в файл sort_results.txt в режиме добавления.

**В классе MainSearch (поиск):**
- После выполнения поиска пользователю предлагается сохранить найденные автомобили.
- Вызов FileSaver.saveSearchResults() передает название файла, результаты поиска, тип поиска и поисковый запрос.
- Результаты сохраняются в файл search_results.txt в режиме добавления.

Сами файлы сохраняются в корневой директории проекта.
