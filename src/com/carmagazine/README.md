# Описание проекта

Данная программа реализует класс `Automobile` (Автомобиль) на языке Java. Класс описывает автомобиль с основными характеристиками и использует паттерн проектирования Builder для удобного и безопасного создания объектов.

Программа также включает:
- валидацию вводимых данных;
- методы `equals()` и `hashCode()`;
- простые тесты без использования JUnit.

## Поля класса Automobile

Класс содержит следующие поля (все — `final` — объект неизменяемый):

- `power` — мощность автомобиля (int)
- `year` — год выпуска (int)
- `configuration` — комплектация автомобиля (String)
- `color` — цвет автомобиля (String)

## Паттерн Builder

Для создания объекта используется вложенный класс `Builder`.

Пример создания автомобиля:

```java
Automobile car = new Automobile.Builder()
        .setPower(150)
        .setYear(2020)
        .setConfiguration("Luxury")
        .setColor("Black")
        .build();
```

Преимущества:
- пошаговое создание объекта;
- проверка данных при установке;
- предотвращение создания некорректных объектов.

## Валидация данных

Проверка выполняется при вызове setter-методов `Builder`.

Проверки:
- **Мощность:** больше 0.
- **Год выпуска:** в диапазоне 1886–2100.
- **Комплектация:** не `null`, не пустая строка.
- **Цвет:** не `null`, не пустая строка.

При некорректных данных выбрасывается `IllegalArgumentException`.  
Если при вызове `build()` не заполнены обязательные поля — выбрасывается `IllegalStateException`.

## Методы equals() и hashCode()

Реализованы для корректного сравнения объектов.  
Два объекта считаются равными, если совпадают `power`, `year`, `configuration` и `color`.  
Это позволяет использовать объекты в `HashSet`, `HashMap` и других коллекциях Java.

## Простые тесты

Тестирование реализовано без JUnit, через класс с `main()`.

Проверяемые случаи:
- создание корректного автомобиля;
- неверная мощность;
- неверный год;
- пустая комплектация;
- пропущенное обязательное поле.

Пример вывода:

```
testValidCar: OK
testInvalidPower: OK
testInvalidYear: OK
testEmptyConfiguration: OK
testMissingField: OK
```


## Используемые технологии

- Java
- Паттерн проектирования Builder
- Базовая валидация данных
- Переопределение `equals()` и `hashCode()`
