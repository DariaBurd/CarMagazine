# CarMagazine: Модуль Меню (Menu System)

Этот пакет содержит классы, отвечающие за пользовательский интерфейс, навигацию по меню и управление потоками выполнения в приложении **CarMagazine**. Модуль предоставляет консольный интерфейс с цветовой схемой, поддержку многопоточности для выполнения задач и систему состояний для навигации.

## 📁 Структура пакета

Пакет: `com.carmagazine.menu`

| Файл | Описание |
|------|----------|
| `MenuManager.java` | Главный контроллер меню. Управляет жизненным циклом приложения, инициализирует зависимости и запускает основной цикл обработки ввода. |
| `MenuHandler.java` | Обработчик логики. Отвечает за переходы между состояниями меню (State Machine) и маршрутизацию пользовательского ввода к соответствующим действиям. |
| `MenuState.java` | Перечисление (enum), определяющее все возможные состояния меню (Главное, Поиск, Сортировка, Источник и т.д.). |
| `MenuPrinter.java` | Отвечает за отображение интерфейса (вывод меню, таблиц, цветов) и выполнение бизнес-логик (поиск, сортировка, загрузка файлов, тесты). |

## 🚀 Функциональность

### 1. Навигация и Интерфейс
*   **Цветовой интерфейс:** Использование ANSI-кодов для подсветки текста (заголовки, пункты меню, ошибки).
*   **Меню состояний:**
    *   **Главное меню:** Выбор основных режимов работы.
    *   **Источник данных:** Загрузка из файла, ручной ввод или генерация случайных автомобилей.
    *   **Поиск:** Фильтрация автомобилей по цвету, комплектации, мощности или году.
    *   **Сортировка:** Сортировка списка автомобилей по различным критериям (включая кастомную сортировку для учебного задания).
*   **Валидация ввода:** Проверка корректности выбора пользователя и вывод сообщений об ошибках.

### 2. Управление данными
*   **Хранение:** Взаимодействие с singleton `CarStorage` для хранения коллекции автомобилей.
*   **Импорт/Экспорт:**
    *   Загрузка данных из `.txt` файлов.
    *   Выгрузка текущей базы автомобилей в файл.
    *   Сохранение результатов поиска и сортировки.
*   **Генерация:** Создание случайных записей об автомобилях.

### 3. Многопоточность
*   Интеграция с `ThreadManager` для выполнения тяжелых задач (поиск, сортировка, загрузка) в отдельном потоке, чтобы не блокировать интерфейс.
*   Возможность включения/выключения режима многопоточности через меню.

### 4. Тестирование
*   Встроенный запуск наборов тестов (проверка валидности автомобилей, тесты поиска, тесты сортировки).

## 🛠 Зависимости

Для корректной работы модуля требуются следующие внешние классы/пакеты (судя по импортам):

*   `com.carmagazine.thread.ThreadManager` — Управление пулом потоков.
*   `com.carmagazine.catalog.CarCollection`, `CarStorage` — Хранилище данных.
*   `com.carmagazine.automobile.Automobile` — Модель данных автомобиля.
*   `com.carmagazine.search.SearchManager` — Логика поиска.
*   `com.carmagazine.sort.*` — Стратегии сортировки.
*   `com.carmagazine.fileSaver.FileSaver` — Работа с файловой системой.
*   `com.carmagazine.test.*` — Классы для тестирования.

## 💻 Использование

Основная точка входа — класс `MenuManager`.

```java
public class Main {
    public static void main(String[] args) {
        MenuManager menuManager = new MenuManager();
        menuManager.start();
    }
}
```

1.  При запуске инициализируется `Scanner`, `ExecutorService` и менеджеры.
2.  Запускается бесконечный цикл `while (isRunning)`, который отображает текущее меню и ожидает ввод пользователя.
3.  Ввод обрабатывается в `MenuHandler`, который меняет состояние `MenuState` или вызывает действия через `MenuPrinter`.
4.  При выборе выхода (пункт "0") выполняется корректное завершение работы (`shutdown`), закрытие потоков и сканера.
